# Trained

A tiny dark-mode phone app that tells Mark what he trained last, so he picks
something different today. Not a workout logger — no sets, reps, exercise
weights, or plans. Just: category cards sorted stalest-first, a rest timer,
and a bodyweight log with a chart.

**Live app:** https://marktran4.github.io/trained/

## Put it on your phone

1. Open the link above in Safari.
2. Share button → **Add to Home Screen**.
3. Open it from the home screen icon — it runs full-screen like a normal app.

## What it does

- **Home** — one card per training category (Push/Pull/Legs/Full Body/Cardio
  by default), showing days since you last trained it, sorted stalest-first.
  Tap a card to log today (or yesterday) in two taps. A week strip and
  28-day dot grid show recent activity at a glance.
- **Timer** — 60/90/120s presets or a custom time. Big countdown, wake lock
  keeps the screen on while running, beeps at zero. iOS can't sound an alarm
  when the screen is locked, so keep it awake and in view.
- **Weight** — log today's kg (overwrites if you log twice in a day), see a
  simple trend chart and recent entries.
- **Settings** (gear icon) — rename/add/delete categories (cap of 8;
  deleting keeps past history), and export your raw data as JSON to the
  clipboard.

## Dev notes

- One static page: `index.html` (all CSS/JS inline, no framework, no build
  step), `sw.js` offline shell (cache-first), `manifest.webmanifest` + icons
  for the home-screen install.
- All data lives in a single `localStorage` key (`trained.v1`). No accounts,
  no sync, no analytics, no network calls beyond loading the page itself.
- Icons are generated by `../make_icons.py` (pure stdlib, no PIL).
- Source of truth lives in the `ea-automation` repo at
  `projects/gym-tracker/`; this repo is just the GitHub Pages deploy target.
